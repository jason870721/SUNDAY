# prd-ticket 對 Sunday 開功能需求票（docs/prd/，全隊通用格式）

覺得 Sunday **缺端點、缺數據、某個 API 該優化**（更多指標、更細的風險視角、結構化 feed、批次操作），就開一張 PRD 票。Sunday 是專門為這支團隊打造的——儘管提，後續會有人實作。

## 怎麼開

1. 先 `read` 掃一眼 `docs/prd/` 現有的 `PRD-*.md`，確認沒有重複、並取**下一個流水號**。
2. `write` 新檔 `docs/prd/PRD-<編號>-<簡短-kebab-描述>.md`（例：`PRD-003-liquidation-feed.md`）。

## 票的格式（三節，缺一不可）

```markdown
# PRD-<編號> — <一句話標題>

## 1. 卡在哪（問題）
你在什麼工作裡、被什麼擋住／看不到什麼。給具體場景，不要抽象抱怨。

## 2. 期望的 API 長相
端點、方法、輸入參數、輸出範例（JSON 給樣子）。寫到「實作者不用再來問你」的程度。

## 3. 為什麼有助於 10% 月目標
這個能力會改變哪個決策／省掉哪段風險／補上哪個盲點。

— <你的名字>, <YYYY-MM-DD>（日期用喚醒訊息的 currenttime）
```

## 紀律

- **一票一事**：兩個需求開兩張票。
- 開完票不用等——回到原本的工作；票是非同步管道，實作者會看。
- 值得 friday 知道的（影響當前交易的缺口）順手 `send_message` 提一句票號。
